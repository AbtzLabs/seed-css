Seed CSS

A light-weight, mobile first and responsive CSS boilerplate.

DOWNLOAD v.0.0.0

(This page design is proudly based on seed-css)


Why you're gonna love it?

It's made for modern browsers, not intrusive and lightweight.


Easy to start

Get started to your projects with a very small footprint CSS boilerplate like this.


Light-weight

Just 11kb of CSS your app needs to load for a mobile first and responsive design.


Based on Sass

All written on Sass, what means you can easily customize it as much as you need.

Get started

Just before get started, please notice that seed-css was written with modern browsers in mind. That means it just has support for IE9+ (which actually isn't a real modern browser ... but), Chrome, Firefox and Safari, besides mobile browsers. If you need support older browsers, this boilerplate may not help you, but, just in case, you're free to customize it and create your own CSS rules to make it supports those older browser versions.

The most first thing you need to do to start using seed-css is import its CSS file into your pages. In order to get the right files, you can either fork this project on Github, or just download the compressed project files to your computer.

After get the latest version of files, import it to your pages as the example below: 

<head >
  ...
  <link href="seed.css" rel="stylesheet" type="text/css" >
</head>

Once you have the seed-css stylesheet embeded to your site or application, have a look on all code examples below to get inspired about what features it offers you and what you can do with it. :)

Grid

Grids are commonly used to define "grid" based designs. The grid system is responsive, which means you can define the number of columns according to the screen size and it will adapts its width to best fit your content.

Offsets are also available. To apply offsets, just add the sufix -offset to your classname.

.md-6 .lg-3
.md-6 .lg-3
.md-6 .lg-3
.md-6 .lg-3
<!-- This's a grid -->
<div class="grid" >
  <!-- This's a row. You can have as many rows as you need. -->
  <div class="row" >
    <!-- This's a column. You can have up to 12 columns in a row. -->
    <div class="col sm-12 md-6 lg-3" > ... </div>
    <div class="col sm-12 md-6 lg-9" > ... </div>
  </div>
</div>
Class Detail Remarks
.grid   Defines the grid container
.row   Defines each row of a grid container
.col   Defines each column of a grid container
.sm-* (1 ~ 12) Defines a column size for small screens. It's used concatenated to .col
.md-* (1 ~ 12) Defines a column size for medium screens. It's used concatenated to .col
.lg-* (1 ~ 12) Defines a column size for large screens. It's used concatenated to .col
.offset-* (1 ~ 12) Creates an offset of columns. It's used concatenated to .col

Buttons

Buttons are a really basic elements on any web page or app, and it comes in a couple of different flavors in seed-css. In order to use the primary just add the class name .btn-primary. If you need use anchor (<a />) as button, include the .btn class name on it.

<!-- .btn class makes an anchor looks like a button --> 
<a class="btn" href="..." > ... </a> 

<!-- default buttons and input[type="button"] -->
<button type="button" > ... </button> 
<input type="button" value="..." />

<!-- .btn-primary makes buttons colored -->
<a class="btn btn-primary" href="..." > ... </a> 
<a class="btn btn-red" href="..." > ... </a> 
<a class="btn btn-green" href="..." > ... </a> 

<!-- .btn-primary makes buttons on different sizes -->
<a class="btn btn-medium" href="..." > ... </a> 
<a class="btn btn-small" href="..." > ... </a> 

<!-- .btn-primary makes disabled buttons -->
<a class="btn disabled" href="..." > ... </a> 
<button disabled > ... </button>

Class Detail Remarks
.btn   Makes anchors looks like buttons
.btn-primary   Defines a 'primary' (blue) colored button
.btn-(*color*)   Defines a colored button. Colors can be: red, orange, green or purple
.btn-(*size*)   Defines a sized button. Sizes can be: small or medium. Default is big
.disabled   Defines disabled buttons. Simply setting the disabled property will also make button looks like disabled
.flat-btn   Defines flat buttons, which means no borders or background colors. You are still able to use .btn-(*color*) and or .btn(*size*) on this button type

Text transform

This text should not break when the screen width is smaller then the text length. 

<p class="nowrap" > ... </p>

this text should be camel case. 

<p class="camel-case" > ... </p>

only the first character of this text should be capitalized. 

<p class="uc-first" > ... </p>

Class Remarks
.nowrap Forces the HTML to ignore white spaces when wrapping lines.
.camel-case Capitalize the first character of any word
.uc-first Capitalize the first character only from the first word
.upper-case Capitalize all character
.lower-case Make all letters in lower case

Hiding

Show and hide elements from your pages can be an usefull ability to improve the user experience of your apps or websites. 

<!-- Hide on all devices --> 
<div class="hide" > ... </div>

<!-- Hide on small devices --> 
<div class="hide-only-small" > ... </div>

<!-- Hide on medium and small devices --> 
<div class="hide-on-medium-and-down" > ... </div>

<!-- Hide on medium and large devices --> 
<div class="hide-on-medium-and-up" > ... </div>
Class Remarks
.hide Hide an element in any device.
.hide-only-small Hide an element only on small devices only.
.hide-on-medium-and-down Hide an element on all devices, except for large ones
.hide-on-medium-and-up Hide an element on all devices, except for small ones

Icons

We're using Ionic Icon Fonts as our set of icons. It's free and can be redistributed as well as seed-css.

Just to keep the flexibility, the icon set is not included on the seed-css stylesheet files. You should import it from an external source. If you would like to keep up to date to the latest versions from Ion icons, we strongly recommend you to get it from Ionic CDN.

In order to use those icons, add the following line to the HEAD tag of any page you're using seed-css:

Go to IonIcons
